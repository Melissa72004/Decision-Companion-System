# RESEARCH_LOG.md

## 1. Objective of Research

The purpose of research during this project was to:
- Understand structured decision-making systems
- Learn how to design rule-based logic
- Explore alternative AI-based approaches
- Implement clean and maintainable project architecture

---

## 2. AI Prompts Used

The following AI prompts were used during development:

- "How to build a decision support system using Python?"
- "How to structure rule-based logic in Python?"
- "Best way to validate user input in Python?"
- "How to handle edge cases in decision logic?"
- "How to structure a GitHub README professionally?"
- "Difference between rule-based system and machine learning model"
- "How to separate logic into modular functions?"

AI suggestions were used as guidance and were reviewed before implementation.

---

## 3. Google Search Queries

The following search queries were used:

- Decision support system architecture
- Rule-based decision logic examples
- Python input validation techniques
- Software design trade-offs examples
- Handling boundary values in conditional logic
- Best practices for structuring Python projects

These searches helped refine system structure and logic implementation.

---

## 4. References Consulted

- Python Official Documentation
- GitHub Markdown Documentation
- Articles on Decision Support Systems
- Software Engineering design principles
- Online tutorials on structured conditional logic

These references influenced project structure and documentation formatting.

---

## 5. What Was Accepted from AI Suggestions

- Modular code structure
- Input validation improvements
- Clear separation between input, processing, and output
- Documentation formatting guidelines
- Edge case handling strategies

These were adapted and modified according to project needs.

---

## 6. What Was Rejected or Modified

Rejected:
- Complex machine learning-based implementation (due to limited dataset)
- Overly advanced architecture not required for project scope
- Unnecessary external libraries

Modified:
- AI-generated logic was simplified for clarity
- Some suggested condition structures were optimized
- Documentation content was rewritten to match actual implementation

---

## 7. Ethical and Transparency Note

AI tools were used for guidance and idea refinement. 
All final logic, structure, and implementation decisions were manually reviewed, tested, and implemented by the developer.
